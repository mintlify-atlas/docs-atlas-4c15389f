## POST /api/completions

Submit a completion for a challenge. The server verifies that the provided GPS
coordinates are within an allowed distance of the challenge location before
recording the completion. Aura points and streak are updated automatically
inside a database transaction.

<Note>
  Auth required: **Yes** — include `Authorization: Bearer <accessToken>`.
  This endpoint is rate-limited. Each challenge can only be completed once per
  user; a duplicate submission returns `400`.
</Note>

### Request body

<ParamField body="challengeId" type="integer" required>
  ID of the challenge being completed.
</ParamField>

<ParamField body="latitude" type="number" required>
  User's current latitude (`-90` to `90`).
</ParamField>

<ParamField body="longitude" type="number" required>
  User's current longitude (`-180` to `180`).
</ParamField>

<ParamField body="imageUrl" type="string" required>
  URL of the proof photo for this completion. Must be a valid URL.
</ParamField>

<ParamField body="caption" type="string">
  Optional caption for the completion post. Maximum 500 characters.
</ParamField>

### Response — 201 Created

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  `"Challenge completed successfully!"`
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">Completion record ID.</ResponseField>
    <ResponseField name="userId" type="integer">ID of the user who completed the challenge.</ResponseField>
    <ResponseField name="challengeId" type="integer">ID of the completed challenge.</ResponseField>
    <ResponseField name="latitude" type="number">Latitude recorded at completion.</ResponseField>
    <ResponseField name="longitude" type="number">Longitude recorded at completion.</ResponseField>
    <ResponseField name="imageUrl" type="string | null">URL of the proof photo.</ResponseField>
    <ResponseField name="imageUri" type="string">URI of the proof photo (mirrors `imageUrl`).</ResponseField>
    <ResponseField name="caption" type="string | null">Optional caption.</ResponseField>
    <ResponseField name="completedAt" type="string">ISO 8601 timestamp of completion.</ResponseField>
    <ResponseField name="likes" type="integer">Like count (starts at `0`).</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `400`  | Invalid `challengeId`, missing `imageUrl`, GPS coordinates outside the allowed distance from the challenge location, or challenge already completed by this user |
| `401`  | Missing or invalid Bearer token |
| `404`  | Challenge not found |
| `429`  | Rate limit exceeded |

### Example

```bash
curl -X POST http://localhost:3000/api/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <accessToken>" \
  -d '{
    "challengeId": 42,
    "latitude": 35.3005,
    "longitude": -120.6596,
    "imageUrl": "https://<project>.supabase.co/storage/v1/object/public/completion-images/42/photo.jpg",
    "caption": "Made it to the top!"
  }'
```

## GET /api/completions

Return a paginated list of completions. Supports filtering by user, challenge,
and date range. Each completion includes embedded `user` and `challenge` objects.

### Query parameters

<ParamField query="userId" type="integer">
  Filter completions by user ID.
</ParamField>

<ParamField query="challengeId" type="integer">
  Filter completions by challenge ID.
</ParamField>

<ParamField query="startDate" type="string">
  ISO 8601 date-time. Return only completions at or after this timestamp.
  Must be before or equal to `endDate` if both are supplied.
</ParamField>

<ParamField query="endDate" type="string">
  ISO 8601 date-time. Return only completions at or before this timestamp.
</ParamField>

<ParamField query="page" type="integer">
  Page number. Defaults to `1`.
</ParamField>

<ParamField query="limit" type="integer">
  Items per page. Defaults to `20`, range `1`–`100`.
</ParamField>

<ParamField query="sortOrder" type="string">
  Sort order for `completedAt`. One of `asc` or `desc`. Defaults to `desc`.
</ParamField>

### Response — 200 OK

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="data" type="array">
  <Expandable title="completion properties">
    <ResponseField name="id" type="integer">Completion ID.</ResponseField>
    <ResponseField name="userId" type="integer">User ID.</ResponseField>
    <ResponseField name="challengeId" type="integer">Challenge ID.</ResponseField>
    <ResponseField name="latitude" type="number">Latitude at completion.</ResponseField>
    <ResponseField name="longitude" type="number">Longitude at completion.</ResponseField>
    <ResponseField name="imageUrl" type="string | null">Proof photo URL.</ResponseField>
    <ResponseField name="imageUri" type="string">Proof photo URI.</ResponseField>
    <ResponseField name="caption" type="string | null">Optional caption.</ResponseField>
    <ResponseField name="completedAt" type="string">ISO 8601 completion timestamp.</ResponseField>
    <ResponseField name="likes" type="integer">Like count.</ResponseField>
    <ResponseField name="user" type="object">
      <Expandable title="properties">
        <ResponseField name="id" type="integer">User ID.</ResponseField>
        <ResponseField name="name" type="string">Display name.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="challenge" type="object">
      <Expandable title="properties">
        <ResponseField name="id" type="integer">Challenge ID.</ResponseField>
        <ResponseField name="title" type="string">Challenge title.</ResponseField>
        <ResponseField name="pointsReward" type="integer">Points reward.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="properties">
    <ResponseField name="page" type="integer">Current page.</ResponseField>
    <ResponseField name="limit" type="integer">Items per page.</ResponseField>
    <ResponseField name="total" type="integer">Total matching completions.</ResponseField>
    <ResponseField name="totalPages" type="integer">Total pages.</ResponseField>
  </Expandable>
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `400`  | Invalid query parameters (e.g. non-numeric ID, `startDate` after `endDate`, `limit` out of range) |

### Example

```bash
curl "http://localhost:3000/api/completions?userId=7&limit=5&sortOrder=desc"
```

## PATCH /api/completions/:id

Update the caption of an existing completion. Only the owner of the completion
may edit it.

<Note>
  Auth required: **Yes** — include `Authorization: Bearer <accessToken>`. The
  authenticated user must own the completion.
</Note>

### Path parameters

<ParamField path="id" type="integer" required>
  The numeric ID of the completion to update.
</ParamField>

### Request body

<ParamField body="caption" type="string">
  Updated caption text. Maximum 500 characters. Pass `null` or omit to clear
  the caption.
</ParamField>

### Response — 200 OK

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="data" type="object">
  The updated completion, including embedded `user` (`id`, `name`) and
  `challenge` (`id`, `title`, `pointsReward`) objects.
</ResponseField>

### Error responses

| Status | Condition |
|--------|-----------|
| `400`  | `id` is not a valid integer, or `caption` exceeds 500 characters |
| `401`  | Missing or invalid Bearer token |
| `403`  | Authenticated user does not own this completion |
| `404`  | Completion not found |

### Example

```bash
curl -X PATCH http://localhost:3000/api/completions/101 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <accessToken>" \
  -d '{"caption": "Updated: caught the sunrise instead!"}'
```
